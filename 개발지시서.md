

# 📋 개발 지시서 (최종본)
## "우리가 만드는 미디어 세상" — 민주주의와 미디어 체험 웹 앱

> **개발 도구:** Codex / **운영 환경:** 맥미니(Apple Silicon 가정) 로컬 서버 + 포트포워딩
> **단일 포트 `8742`로 프론트+백엔드 통합 운영**

---

## 1. 프로젝트 개요

6학년 사회 "민주주의와 미디어" 단원의 3차시 체험형 수업 도구. 교사가 차시를 제어하고, 학생들은 노트북(1인 1기기)으로 접속해 미디어 탐색 → 댓글 활동 → 여론 조작 역할극 → 추리 및 반성을 수행한다.

**핵심 시나리오:** 가상 도시 "한빛시"의 킥보드 전면 금지 조례를 둘러싼 여론전
- **A조 = 킥보드 업체 "쌩쌩모빌"** → 금지 **반대** 여론 조작
- **B조 = 택시·버스 연합 "한빛교통조합"** → 금지 **찬성** 여론 조작

---

## 2. 기술 스택

| 구분 | 기술 |
|------|------|
| 프론트엔드 | React (Vite) + Tailwind CSS + React Router v6 |
| 백엔드 | Node.js + Express |
| 실시간 통신 | Socket.io |
| 데이터베이스 | SQLite (`better-sqlite3`) |
| 정적 서빙 | Express가 React 빌드물(`dist/`)을 서빙 → 단일 포트 |
| 포트 | **8742** (TCP) |

> ⚠️ **better-sqlite3 빌드 주의 (Apple Silicon):** 네이티브 모듈이라 설치 시 빌드가 막힐 수 있음. 막히면 다음 순서로 대응:
> 1. `xcode-select --install` 로 빌드 도구 설치 확인
> 2. Node 버전을 LTS(20.x 이상)로 맞추기
> 3. 그래도 안 되면 대안으로 `node:sqlite`(Node 22+ 내장) 또는 `sql.js` 사용 검토
> 빌드 단계에서 멈추지 말고 위 대안 중 하나로 진행할 것.

## 5. DB 테이블 설계

```sql
-- 교사
CREATE TABLE teachers (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  class_code TEXT UNIQUE NOT NULL,    -- 예: "6-2"
  password TEXT NOT NULL,
  current_phase INTEGER DEFAULT 0     -- 0:대기 / 1 / 2 / 3 (양방향 이동 가능)
);

-- 학생 명단 (교사가 CSV로 사전 등록 — 이 명단에 있는 학생만 입장 가능)
CREATE TABLE students (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  class_code TEXT NOT NULL,
  student_number TEXT NOT NULL,       -- 번호
  name TEXT NOT NULL,                 -- 이름
  team TEXT CHECK(team IN ('A','B')), -- 3차시 조 배정 (NULL 허용)
  session_token TEXT,                 -- 입장 시 발급, 중복 접속 방지
  is_active BOOLEAN DEFAULT FALSE,    -- 현재 접속 여부
  UNIQUE(class_code, student_number)
);

-- 1차시 응답
CREATE TABLE phase1_responses (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id INTEGER NOT NULL,
  most_accurate TEXT NOT NULL,
  most_emotional TEXT NOT NULL,
  free_text TEXT,
  submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 2차시 댓글 분류
CREATE TABLE phase2_classifications (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id INTEGER NOT NULL,
  comment_id TEXT NOT NULL,
  label TEXT CHECK(label IN ('good','bad')),
  reason TEXT,
  submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 2차시 직접 댓글 (실명)
CREATE TABLE phase2_comments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id INTEGER NOT NULL,
  content TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 3차시 조작 댓글 (화면엔 익명, DB엔 작성자 기록)
CREATE TABLE phase3_comments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id INTEGER NOT NULL,
  team TEXT NOT NULL,                 -- 'A' 또는 'B'
  content TEXT NOT NULL,
  is_revealed BOOLEAN DEFAULT FALSE,  -- 정체 공개 후 TRUE
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 3차시 투표
CREATE TABLE phase3_votes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id INTEGER NOT NULL,
  vote TEXT CHECK(vote IN ('trust','suspicious','unsure')),
  submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(student_id)                  -- 1인 1표
);

-- 3차시 추리 서술 (담벼락)
CREATE TABLE phase3_guesses (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id INTEGER NOT NULL,
  guess_text TEXT NOT NULL,
  is_correct BOOLEAN,                 -- 교사가 정답 체크
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 3차시 반성 서술
CREATE TABLE phase3_reflections (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id INTEGER NOT NULL,
  answer1 TEXT,
  answer2 TEXT,
  answer3 TEXT,
  submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 6. API 설계

### 인증 / 명단
```
POST /api/auth/teacher-login
  body: { class_code, password }
  → 교사 세션 토큰 발급

POST /api/teacher/upload-roster
  body: CSV 파일 (컬럼: 번호, 이름)
  → students 테이블에 일괄 등록 (기존 명단 덮어쓰기 옵션)

POST /api/auth/student-login
  body: { class_code, student_number, name }
  → 명단 대조 (번호+이름 둘 다 일치해야 통과)
  → 이미 is_active면 거부
  → 통과 시 session_token 발급, is_active=TRUE
  → 실패 메시지 구분:
     · 명단에 없음        → "명단에 없는 정보입니다. 선생님께 문의하세요"
     · 번호/이름 불일치    → "이름 또는 번호를 다시 확인해주세요"
     · 이미 접속 중        → "이미 접속된 번호입니다. 선생님께 문의하세요"

GET  /api/auth/phase
  → 현재 활성 차시 반환 (학생 폴링 또는 socket으로 대체 가능)
```

### 교사 전용
```
POST /api/teacher/set-phase        body: { phase }  -- 0~3, 양방향 이동 허용
POST /api/teacher/assign-teams     -- A/B조 자동 배정 (번호 홀짝 또는 랜덤)
GET  /api/teacher/students         -- 전체 명단 + 접속/제출 현황
POST /api/teacher/reset-session    body: { student_id }  -- 특정 학생 강제 재접속 허용
GET  /api/teacher/phase1           -- 1차시 응답 전체
GET  /api/teacher/phase2           -- 2차시 댓글+분류 전체
GET  /api/teacher/phase3           -- 3차시 댓글+투표+추리+반성 전체 (작성자 표시)
POST /api/teacher/reveal           -- 정체 공개 (phase3_comments.is_revealed 일괄 TRUE)
POST /api/teacher/mark-correct     body: { guess_id, is_correct }
```

### 학생 전용
```
GET  /api/student/phase1/content
POST /api/student/phase1/submit

GET  /api/student/phase2/article
POST /api/student/phase2/classify
POST /api/student/phase2/comment

GET  /api/student/phase3/article       -- 본인 팀 언론사 기사 반환
GET  /api/student/phase3/journalist    -- 기자 프로필 (모달용)
POST /api/student/phase3/comment       -- 익명 조작 댓글
GET  /api/student/phase3/board         -- 교차 공개 후 상대 팀 게시판
POST /api/student/phase3/vote
POST /api/student/phase3/guess
POST /api/student/phase3/reflection
```

### Socket.io 이벤트
```
[서버 → 클라이언트]
phase_changed      -- 교사 차시 변경(되돌리기 포함) 시 전체 브로드캐스트
new_comment        -- 새 댓글 실시간 전파
board_revealed     -- 정체 공개 브로드캐스트
vote_updated       -- 투표 집계 실시간 갱신
guess_submitted    -- 담벼락에 새 추리 카드 추가

[클라이언트 → 서버]
submit_comment / submit_vote / submit_guess
```

## 7. 세션 처리 (계정·비번 없음)

```
학생 입장 성공
  → 서버가 랜덤 session_token 발급, DB의 students.session_token 저장
  → 클라이언트 localStorage에 토큰 저장
  → 이후 모든 요청 헤더에 토큰 포함

새로고침 시
  → localStorage 토큰으로 자동 재인증
  → 같은 토큰이면 그대로 입장 유지 (중복 아님)

교사가 reset-session 호출 시
  → 해당 학생 토큰 무효화, is_active=FALSE → 재접속 허용
```

## 10. 서버 진입점 (`server/index.js`) 핵심

```js
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');
const path = require('path');

const app = express();
const server = http.createServer(app);
const io = new Server(server, { cors: { origin: '*' } });
const PORT = 8742;

app.use(express.json());
app.use(express.static(path.join(__dirname, '../client/dist')));

app.use('/api/auth', require('./routes/auth'));
app.use('/api/teacher', require('./routes/teacher'));
app.use('/api/student', require('./routes/student'));

app.get('*', (req, res) =>
  res.sendFile(path.join(__dirname, '../client/dist/index.html')));

io.on('connection', (socket) => {
  socket.on('submit_comment', (d) => io.emit('new_comment', d));
  socket.on('submit_vote',    (d) => io.emit('vote_updated', d));
  socket.on('submit_guess',   (d) => io.emit('guess_submitted', d));
});

server.listen(PORT, '0.0.0.0', () =>
  console.log(`서버 실행: http://localhost:${PORT}`));
```

## 12. 포트포워딩 설정 (교사 직접 — 개발 범위 외)

```
공유기 관리자 → 포트포워딩 규칙 추가
  외부 포트 8742 → 내부(맥미니) IP:8742, 프로토콜 TCP
맥미니 로컬 IP 확인:  터미널 → ipconfig getifaddr en0
학생 접속 주소:       http://[공인IP]:8742
```

> 참고: 공인 IP는 변동될 수 있으니 수업 당일 `curl ifconfig.me`로 확인. 가능하면 DDNS 설정 권장.
