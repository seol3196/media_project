# 미디어 프로젝트 소개

## 1. 한눈에 보는 앱 개요

`우리가 만드는 미디어 세상`은 초등학교 6학년 사회 `민주주의와 미디어` 단원을 체험형 수업으로 운영하기 위한 웹 앱이다. 교사가 수업 진행 단계를 열고 닫으면 학생 화면이 실시간으로 반응하고, 학생들은 각자 노트북이나 태블릿으로 접속해 미디어 자료를 보고, 댓글을 고쳐 쓰고, 댓글 조작 역할극과 추리 활동을 수행한다.

이 앱은 단순히 문제를 푸는 사이트가 아니라, 교사가 전자칠판용 대시보드를 띄워 놓고 학생들과 함께 한 단계씩 수업을 진행하도록 설계되어 있다. 학생은 미리 모든 활동 내용을 알기보다, 교사가 연 활동에 들어가고 교사가 다음 문제나 다음 게시물을 열어 줄 때마다 순서대로 참여한다.

핵심 학습 목표는 다음과 같다.

- 같은 사건도 매체 형식에 따라 다르게 전달될 수 있음을 이해한다.
- 기사, 방송, SNS, 유튜브 게시물을 비교하며 정보의 전달 방식과 특징을 구분한다.
- 댓글이 토론을 돕거나 방해할 수 있음을 실제 댓글 수정 활동으로 경험한다.
- 댓글 조작이 사람들의 판단과 분위기에 영향을 줄 수 있음을 역할극과 추리 활동으로 체험한다.
- 온라인에서 정보를 볼 때 출처, 표현 방식, 감정 자극, 댓글 흐름을 비판적으로 살펴보는 태도를 기른다.

## 2. 사용자 역할

이 앱에는 세 가지 사용자 역할이 있다.

### 관리자

관리자는 교사가 사용할 반 코드를 발급한다. 로그인 화면에서 `관리자` 탭을 선택하고 관리자 계정으로 들어가면 반 코드를 만들 수 있다.

- 기본 관리자 아이디: `admin`
- 기본 관리자 비밀번호: `se2519925!`

관리자가 반 코드를 발급하면 교사는 별도 비밀번호 없이 해당 반 코드로 교사 페이지에 들어간다.

### 교사

교사는 수업을 운영하는 중심 사용자다. 교사 페이지에서는 다음 작업을 한다.

- 학생 명단 등록
- 학생 번호, 이름, 모둠 수정
- 학생별 활동 진행 현황 확인
- 활동1, 활동2, 활동3 열기/닫기
- 활동별 전자칠판용 대시보드 열기
- 활동별 진행 단계 제어
- 활동별 또는 전체 활동 초기화
- 학생별 활동 기록 초기화
- 선택 학생 일괄 삭제
- 활동3 모둠 수 설정 및 자동 배정

교사용 활동 대시보드는 보통 새 창으로 열어 전자칠판에 띄우는 용도다. 교사 기본 대시보드는 수업 관리용이고, 활동 대시보드는 학생들과 함께 보는 수업 진행용 화면이다.

### 학생

학생은 교사가 등록한 명단에 있는 번호와 이름으로만 입장할 수 있다. 로그인 후 학생 대시보드에는 열린 활동만 들어갈 수 있게 표시된다. 학생 대시보드에는 활동의 자세한 내용이 미리 보이지 않도록 구성되어 있으며, 수업 중 교사의 진행에 맞춰 활동을 경험한다.

학생은 각 활동에서 다음 일을 한다.

- 활동1: 네 매체를 보고 매체별 전달 방식과 특징을 비교한다.
- 활동1: SNS 게시물을 직접 만들어 우리 반 피드에 올리고 좋아요를 누른다.
- 활동2: 문제가 있는 댓글을 더 좋은 댓글로 고쳐 쓴다.
- 활동2: 다른 학생들이 고친 댓글을 확인한다.
- 활동3: 모둠별 미션에 따라 조작 댓글을 작성한다.
- 활동3: 다른 게시글에 일반 댓글을 달고, 조작 댓글을 추리한다.

## 3. 로그인 화면

로그인 화면은 어두운 배경 위에 뉴스 자막, 댓글, 해시태그가 계속 흘러가는 `정보 폭주` 느낌의 CSS 애니메이션을 사용한다. 영상 파일이나 GIF를 쓰지 않고 순수 CSS 애니메이션만 사용하므로 서버와 회선에 큰 부담을 주지 않는다.

왼쪽에는 수업의 주제를 보여준다.

- 작은 라벨: `민주주의와 미디어`
- 큰 제목: `미디어 세계`
- 부제: `쏟아지는 정보 속에서, 무엇을 믿을 것인가`

오른쪽에는 로그인 카드가 있다. 로그인 카드는 학생, 교사, 관리자 탭으로 나뉜다.

- 학생: 반 코드, 번호, 이름 입력
- 교사: 반 코드 입력
- 관리자: 아이디, 비밀번호 입력

사용자가 운영체제에서 `동작 줄이기` 설정을 켜 둔 경우, 배경 애니메이션은 정지된 텍스트로 보이도록 접근성 처리가 되어 있다.

## 4. 관리자 페이지

관리자 페이지의 목적은 반 코드를 발급하는 것이다. 교사가 여러 학급을 운영할 수 있도록 관리자 페이지에서 새 반 코드를 만들 수 있다.

반 코드가 생성되면 `teachers` 테이블에 저장된다. 교사는 이 반 코드로 교사 페이지에 들어가고, 학생도 같은 반 코드를 입력해 해당 반 명단 안에서 로그인한다.

## 5. 교사 대시보드

교사 대시보드는 수업 준비와 운영을 위한 메인 화면이다.

### 5.1 활동 공개 상태

화면 상단에는 현재 열린 활동 상태가 표시된다. 활동1, 활동2, 활동3은 세로 목록으로 배치되어 있으며, 각 활동마다 다음 버튼이 있다.

- `활동 대시보드`: 전자칠판용 활동 진행 화면을 새 창으로 연다.
- `활동 열기`: 학생 대시보드에서 해당 활동에 접속할 수 있게 한다.
- `활동 닫기`: 학생 대시보드에서 해당 활동을 닫는다.
- `활동 설명 보기`: 교사가 활동 흐름을 이해할 수 있도록 상세 설명을 펼쳐 보여 준다.

활동 설명은 교사가 수업 흐름을 빠르게 이해할 수 있도록 활동별로 다르게 구성되어 있다.

활동1 설명은 매체 비교와 SNS 만들기 흐름을 안내한다.

활동2 설명은 댓글을 읽고, 문제 댓글을 공개한 뒤, 학생들이 더 나은 댓글로 고쳐 쓰는 흐름을 안내한다.

활동3 설명은 댓글 조작 역할극, 일반 댓글 작성, 조작 댓글 추리, 정답 공개 흐름을 안내한다.

### 5.2 활동 초기화

기존에는 활동 카드마다 초기화 버튼이 있었지만, 현재는 상단의 초기화 영역으로 통합되어 있다.

초기화 대상 드롭다운에서 다음 중 하나를 선택한다.

- 활동1
- 활동2
- 활동3
- 모든 활동

선택 후 `초기화` 버튼을 누르면 해당 활동의 학생 작업 내용이 초기화된다. `모든 활동`을 선택하면 모든 활동 기록과 진행 상태가 함께 초기화된다.

### 5.3 활동3 모둠 관리

활동3이 열려 있을 때는 모둠 수 설정과 자동 배정 버튼이 표시된다. 모둠 수는 2개부터 6개까지 선택할 수 있다. 선택한 모둠 수에 따라 학생들이 A~F 모둠으로 자동 배정된다.

### 5.4 명단 등록

교사는 명단 등록 영역에 이름을 한 줄에 한 명씩 붙여 넣을 수 있다. 번호는 입력 순서대로 1, 2, 3, 4처럼 자동으로 붙는다. 등록 후 아래 학생 현황 표에서 번호, 이름, 모둠을 직접 수정할 수 있다.

CSV 파일 선택 방식도 남아 있어, 파일로 명단을 넣는 것도 가능하다.

### 5.5 학생 현황

학생 현황 표에는 다음 정보가 표시된다.

- 선택 체크박스
- 번호
- 이름
- 조
- 접속 상태
- 활동1 제출 여부
- 활동2 참여 여부
- 활동3 제출 여부
- 관리 버튼

학생별 관리 기능은 다음과 같다.

- `수정`: 학생 번호, 이름, 모둠을 수정한다.
- `활동 초기화`: 해당 학생의 모든 활동 기록만 초기화한다.
- 체크박스 선택 후 `선택 학생 삭제`: 선택한 학생들을 일괄 삭제한다. 삭제 시 해당 학생의 활동 기록도 함께 삭제된다.

학생 현황 표 아래에는 예전의 활동1 응답, 활동2 댓글, 활동3 투표/조작댓글/담벼락 모니터 영역이 제거되어 있다. 현재 교사는 필요한 정보와 결과를 각 활동 대시보드 또는 포스트잇 창에서 확인한다.

## 6. 학생 대시보드

학생이 로그인하면 학생 대시보드로 들어간다. 학생 대시보드는 열린 활동만 표시한다. 학생이 활동을 미리 자세히 알고 들어가는 것을 막기 위해 활동 설명은 길게 보여 주지 않는다.

학생 화면 상단의 `우리가 만드는 미디어 세상` 제목을 누르면 학생 대시보드로 돌아간다. 활동 화면 오른쪽 위에 있던 `활동 목록` 버튼은 학생들이 실수로 나가는 일이 많아 제거되어 있다.

## 7. 활동1: 미디어 탐색과 SNS 게시물 만들기

활동1은 같은 사건을 여러 매체 형식으로 비교하는 활동이다. 학생은 신문, 방송, SNS, 유튜브 형식의 게시물을 살펴본다. 각 매체는 단순한 카드가 아니라 실제 매체처럼 보이도록 구성되어 있다.

### 7.1 학생 활동 흐름

활동1은 교사가 전자칠판용 활동1 대시보드에서 문제를 하나씩 진행한다. 학생은 현재 열린 문제만 풀 수 있다.

기본 흐름은 다음과 같다.

1. 네 매체 모두 보기
2. 매체별 전달 방식 체크
3. 매체별 특징 연결
4. 가장 정확해 보이는 매체 선택과 이유 쓰기
5. 가장 관심을 끄는 매체 선택과 이유 쓰기
6. SNS 게시물 만들기

현재 구현에서는 학생이 신문, 방송, SNS, 유튜브를 모두 클릭해야만 문제를 풀 수 있는 제한은 제거되어 있다. 교사가 수업 진행에 맞춰 문제를 열어 주면 학생이 참여한다.

### 7.2 교사용 활동1 대시보드

교사는 활동1 대시보드에서 현재 문제를 보여 주고, 학생들이 문제를 푼 뒤 정답 확인이나 다음 문제 진행을 제어한다.

문제1과 문제2는 처음에는 문제만 보이고, 교사가 `정답확인`을 눌렀을 때 정답이 나온다. 문제2의 선지는 순서대로 답이 되지 않도록 섞여 있다.

문제3은 학생들이 이유나 생각을 작성해 제출하는 형태다. 교사는 `답변 살펴보기`를 눌러 포스트잇 형태의 답변 모음 화면을 볼 수 있다.

문제4는 학생들이 실제 SNS 게시물을 만들어 보는 활동이다.

### 7.3 우리 반 SNS

학생은 사진을 선택하거나 `사진 선택 안 함`을 선택한 뒤 짧은 글을 작성해 게시한다. 게시 후에는 `우리반 SNS 보기` 버튼이 활성화된다.

우리 반 SNS 화면은 세로형 피드로 구성되어 있으며, 다른 학생들이 올린 게시글을 볼 수 있다. 댓글 기능은 사용하지 않고 좋아요만 누를 수 있다.

교사용 활동1 대시보드에서도 우리 반 SNS 피드를 열어 전자칠판으로 볼 수 있다.

## 8. 활동2: 댓글 고쳐 쓰기

활동2는 댓글의 표현 방식이 토론 분위기에 어떤 영향을 주는지 배우는 활동이다. 활동1에서 본 네 가지 게시물이 다시 등장하고, 각 게시물에는 댓글 5개가 달려 있다.

각 게시물 댓글 5개 중 1개는 올바르지 못한 댓글이다. 처음에는 학생 화면에서 어떤 댓글이 문제인지 바로 표시되지 않는다. 교사가 활동2 대시보드에서 `정답보기`를 눌러야 고쳐야 할 댓글이 공개되고, 학생의 수정 기능이 활성화된다.

### 8.1 게시물 단계

활동2는 네 단계로 진행된다.

1. 신문 게시물
2. 방송 게시물
3. SNS 게시물
4. 유튜브 게시물

교사가 다음 게시물로 넘어가야 학생 화면도 다음 게시물을 볼 수 있다. 교사는 이전 게시물로 돌아갈 수도 있다.

### 8.2 학생 활동 흐름

학생은 게시물과 댓글을 읽는다. 교사가 정답을 공개하면 문제가 있는 댓글이 표시된다. 학생은 수정 버튼을 눌러 해당 댓글을 더 좋은 댓글로 바꿔 제출한다.

제출이 완료되면 다른 학생 댓글 보기가 활성화된다. 학생은 다른 학생들이 고친 댓글을 볼 수 있다.

### 8.3 교사용 활동2 대시보드

교사는 활동2 대시보드에서 다음 작업을 한다.

- 현재 게시물 보기
- 정답보기
- 이전 게시물 / 다음 게시물 이동
- 수정된 댓글 보기

`수정된 댓글 보기`를 누르면 새 창에서 포스트잇 형태의 댓글 모음 화면이 열린다. 원래 활동2 대시보드는 그대로 유지되므로, 교사는 전자칠판용 대시보드에서 다음 학습을 계속 진행할 수 있다.

포스트잇 댓글보기 창에는 다음 요소가 있다.

- 해당 게시물 제목
- 제출 개수
- 창닫기 버튼
- 예시 답안 포스트잇
- 학생들이 제출한 수정 댓글 포스트잇

예시 답안은 답안지 문장이 아니라 실제 학생들이 쓸 법한 모범 댓글 톤으로 작성되어 있다.

## 9. 활동3: 댓글전쟁

활동3은 댓글 조작 역할극과 추리 활동이다. 학생들은 모둠별로 서로 다른 게시글을 맡고, 숨은 이해관계를 가진 인물 역할을 수행한다. 목표는 자신의 정체를 들키지 않으면서 특정 방향의 댓글 분위기를 만드는 것이다.

활동 제목은 `댓글전쟁`이다. 기존의 `여론전쟁`, `여론조작` 같은 표현은 학생 눈높이와 활동 성격에 맞춰 `댓글전쟁`, `댓글조작`으로 바뀌어 있다.

### 9.1 게시물 구성

활동3 게시물은 A~F까지 최대 6개다. 교사가 설정한 모둠 수에 따라 일부 게시물만 사용된다.

각 게시물은 서로 다른 플랫폼 느낌을 갖는다.

- A: SNS/인스타그램 느낌의 마라탕 급식 게시물
- B: 실시간 SNS 느낌의 교복 자율화 게시물
- C: 유튜브 느낌의 피구 폐지 논란 게시물
- D: 커뮤니티 게시판 느낌의 칭찬 스티커 제도 게시물
- E: 뉴스 기사 느낌의 가을 축제 방향 게시물
- F: 온라인 청원 느낌의 무료 우유급식 게시물

각 게시물에는 해당 내용에 어울리는 사진이 최소 한 장씩 들어간다. 사진은 복잡한 AI 이미지 느낌을 피하기 위해 단순한 구도, 낮은 채도, 실제 학교 사진 같은 분위기로 구성되어 있다.

### 9.2 교사용 활동3 대시보드

활동3은 교사용 대시보드의 단계 제어가 중요하다. 대시보드에는 다음 단계가 있다.

1. 설명 화면
2. 댓글조작 시작
3. 모둠별 활동 현황
4. 일반 댓글 달기 진행 현황
5. 조작 댓글 찾기
6. 정답 공개

교사는 설명 화면에서 학생들에게 활동 규칙을 안내한다. 마지막 안내 후 `댓글조작 시작하기`를 누르면 학생 화면에 모둠별 게시글과 미션 쪽지가 열린다.

### 9.3 학생 활동 흐름

학생은 자신의 모둠에 배정된 게시글을 본다. 미션 쪽지를 열면 자신이 어떤 이해관계를 가진 사람인지 알 수 있다. 학생은 그 미션에 맞는 조작 댓글을 작성한다.

조작 댓글 작성 후 준비완료를 누르면 교사 화면에서 모둠별 진행 상황이 표시된다. 교사가 다음 단계로 넘기면 학생은 다른 모둠 게시물에 일반 댓글을 단다.

일반 댓글 단계에서 학생에게 보이는 게시물 목록은 `게시물 1`, `게시물 2`처럼만 표시된다. 어떤 게시물인지 미리 알 수 없도록 목록에서는 사진과 플랫폼 정보가 숨겨진다. 게시물 안으로 들어가야 실제 게시물과 사진을 볼 수 있다.

마지막으로 교사가 조작 댓글 찾기 단계를 열면 학생들은 게시물별 댓글 중 조작 댓글로 의심되는 댓글을 선택한다. 자신의 모둠이 조작한 게시물에서는 점수를 얻지 못하도록 되어 있어, 학생은 다른 모둠의 댓글을 잘 추리해야 한다.

### 9.4 정답 공개

교사가 정답을 공개하면 학생은 자신의 점수를 확인하고, 각 게시물에서 어떤 댓글이 조작 댓글이었는지 볼 수 있다. 교사용 화면에서는 모둠별 숨은 이권과 미는 방향을 확인할 수 있다.

## 10. 실시간 동기화 방식

이 앱은 Socket.io를 사용해 교사 화면과 학생 화면을 실시간으로 연결한다.

예를 들어 교사가 활동1에서 다음 문제로 넘어가면 서버는 `activity1_state_changed` 이벤트를 보낸다. 학생 화면은 이 이벤트를 듣고 있다가 현재 문제를 새로 불러온다.

주요 실시간 이벤트는 다음과 같다.

- `phase_status_changed`: 어떤 활동이 열렸는지 학생 대시보드에 반영
- `activity1_state_changed`: 활동1 현재 문제와 정답 공개 상태 변경
- `activity1_padlet_updated`: 활동1 답변 포스트잇 갱신
- `activity1_sns_updated`: 우리 반 SNS 피드 갱신
- `activity2_state_changed`: 활동2 현재 게시물과 정답 공개 상태 변경
- `activity2_revisions_updated`: 활동2 수정 댓글 모음 갱신
- `activity3_updated`: 활동3 단계, 댓글, 선택 상태 갱신

학생은 페이지를 새로고침하지 않아도 교사의 진행에 따라 화면이 바뀐다. 다만 소켓은 화면 상태를 알려 주는 역할이고, 실제 데이터는 API로 다시 불러오는 구조다.

## 11. 데이터 저장 구조

데이터베이스는 SQLite를 사용한다. DB 파일은 `server/db/media.db`에 생성된다.

주요 테이블은 다음과 같다.

- `teachers`: 반 코드, 열린 활동, 활동별 진행 상태 저장
- `students`: 학생 명단, 번호, 이름, 모둠, 접속 상태 저장
- `activity1_answers`: 활동1 문제별 답변 저장
- `activity1_sns_posts`: 활동1 학생 SNS 게시글 저장
- `activity1_sns_likes`: 활동1 우리 반 SNS 좋아요 저장
- `activity2_comment_revisions`: 활동2 학생 수정 댓글 저장
- `phase3_comments`: 활동3 조작 댓글과 일반 댓글 저장
- `phase3_team_ready`: 활동3 모둠 준비 상태 저장
- `phase3_selections`: 활동3 조작 댓글 추리 선택 저장
- `phase3_votes`, `phase3_guesses`, `phase3_reflections`: 예전 흐름과 일부 기록용 테이블

`media.db`는 `.gitignore` 처리되어 있어 GitHub에 올라가지 않는다. 따라서 로컬에서 테스트로 넣은 댓글이나 학생 데이터는 코드와 별개로 현재 컴퓨터의 DB에만 남는다.

## 12. 실행 방법

개발 중에는 다음 명령을 사용한다.

```bash
npm run dev
```

이 명령은 프론트엔드 Vite 서버와 백엔드 Express 서버를 동시에 실행한다.

- 프론트 개발 서버: `http://localhost:5173`
- 백엔드 서버: `http://localhost:8742`

수업 배포용으로 빌드할 때는 다음 명령을 사용한다.

```bash
npm run build
npm start
```

`npm run build`는 React 앱을 `dist/`로 빌드한다. `npm start`는 Express 서버를 실행한다. Express 서버는 API와 정적 파일 서빙을 담당한다.

- `/teacher/activity1-sns`: 활동1 우리 반 SNS 보기
- `/teacher/activity2-board`: 활동2 전자칠판용 대시보드
- `/teacher/activity2-revisions`: 활동2 수정 댓글 포스트잇 보기

## 14. 수업 운영 예시

수업 전 준비 흐름은 다음과 같다.

1. 관리자가 반 코드를 발급한다.
2. 교사가 반 코드로 로그인한다.
3. 교사가 학생 명단을 등록한다.
4. 필요한 경우 활동3 모둠 수를 정하고 자동 배정한다.
5. 교사가 활동1을 열고 활동1 대시보드를 전자칠판에 띄운다.

수업 중 활동1 흐름은 다음과 같다.

1. 교사가 문제1을 보여 준다.
2. 학생들이 문제1을 푼다.
3. 교사가 정답을 확인한다.
4. 다음 문제로 넘어간다.
5. 학생들이 SNS 게시물을 작성한다.
6. 교사가 우리 반 SNS를 열어 함께 본다.

활동2 흐름은 다음과 같다.

1. 교사가 활동2를 열고 활동2 대시보드를 띄운다.
2. 게시물 1을 함께 본다.
3. 교사가 정답보기를 눌러 고쳐야 할 댓글을 공개한다.
4. 학생들이 댓글을 고쳐 쓴다.
5. 교사가 수정된 댓글 보기를 새 창으로 열어 포스트잇을 함께 본다.
6. 다음 게시물로 넘어가 같은 흐름을 반복한다.

활동3 흐름은 다음과 같다.

1. 교사가 활동3을 열고 활동3 대시보드를 띄운다.
2. 활동 설명을 진행한다.
3. 학생들이 미션에 맞는 조작 댓글을 작성한다.
4. 학생들이 다른 게시물에 일반 댓글을 단다.
5. 학생들이 조작 댓글을 찾는다.
6. 교사가 정답을 공개하고 숨은 이권을 설명한다.

## 15. 초기화와 삭제 기능

교사는 수업 중 필요에 따라 기록을 초기화할 수 있다.

활동 초기화는 활동1, 활동2, 활동3, 모든 활동 중 선택해서 실행한다. 해당 활동의 학생 작업 내용과 활동 진행 상태가 초기화된다.

학생별 활동 초기화는 특정 학생의 활동 기록만 지운다. 학생 명단 자체는 유지된다. 학생이 잘못 들어갔거나 테스트 계정으로 활동한 경우에 유용하다.

선택 학생 삭제는 학생 현황 표에서 체크한 학생들을 명단에서 삭제한다. 삭제 시 그 학생들의 활동 기록도 함께 삭제된다.

## 16. 설계상 중요한 점

이 앱은 교사 주도 수업을 전제로 한다. 학생이 모든 콘텐츠를 자유롭게 탐색하는 구조가 아니라, 교사가 단계별로 열어 주고 학생 화면이 그에 맞춰 따라가는 구조다.

활동별 대시보드는 교사가 전자칠판에 띄우기 좋도록 별도 창으로 열린다. 포스트잇 보기나 우리 반 SNS 보기 역시 별도 창으로 띄울 수 있어, 원래 활동 대시보드를 유지한 채 학생 답변만 따로 확인할 수 있다.

또한 영상 파일을 사용하지 않는 방향으로 설계되어 있다. 로그인 화면의 정보 폭주 효과는 CSS 텍스트 애니메이션이고, 활동 자료도 이미지와 HTML/CSS 기반 UI로 구성되어 서버와 네트워크 부담을 줄인다.

## 17. 현재 앱의 교육적 핵심

이 앱이 다루는 핵심은 “정보를 많이 보는 것”이 아니라 “정보가 어떻게 보이도록 만들어지는지 알아차리는 것”이다.

학생은 같은 사건이 신문, 방송, SNS, 유튜브에서 다르게 느껴지는 이유를 경험한다. 또 댓글 하나가 토론을 더 낫게 만들 수도 있고, 싸움이나 오해를 키울 수도 있음을 직접 고쳐 쓰며 배운다. 마지막으로 활동3에서는 댓글 조작 역할을 직접 해 보고, 다른 학생의 댓글을 추리하면서 온라인 여론과 댓글 흐름을 비판적으로 바라보는 힘을 기른다.

따라서 이 앱은 미디어 리터러시, 민주적 의사소통, 온라인 시민성 교육을 함께 다루는 수업 도구라고 볼 수 있다.
